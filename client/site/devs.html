What is Paperbots?

Paperbots is an educational programming environment running in the browser. It was created to enable teaching or self-learning of computational thinking and programming.

Paperbots consists of:

  • The Wee programming language, compiler and virtual machine.
  • The Paperbots website, where users can
    • Create an account.
    • Create programs for different micro worlds in a browser based IDE.
    • Save programs to their account.
    • Share programs with others.
  • The Paperbots course guiding beginner's through learning computational thinking and programming.

Why Paperbots?

The foremost goal of Paperbots is to make it easier to learn (and teach) computational thinking and programming.

There are many educational programming environments, like Scratch, commonly in use in computing education. Scratch is a great way to give kids up until the age of 12 a taste of computing. However, past that age, Scratch is viewed as being "for kids" and "not being the real deal". Young adults want something that is closer to the "real world". It is also not immediately clear how to go from Scratch to a "real" programming environment.

In that real world, "professional" programming languages, like Python or Lua, are touted as being great for beginners. While these languages are indeed great, their surrounding programming environments and ecosystems are very complex. It is non-trivial to get setup, and the development environments are overwhelming. It is also very hard to share programs you've created with these tools.

With Paperbots, I'm trying to build a bridge between the world of Scratch and professional programming environments.

I know how to program, why should I care?

Because Paperbots is not only an educational tool, but also a recreational tool. If you are looking for a simple environment to quickly whip up and share ideas for small games or generative art, Paperbots might be for you.

As a side-effect, your programs can serve as inspiration for beginners! And maybe yourself have kids or friends who'd like to get a glimpse into computational programming. Maybe Paperbots can be their pathway into that particular rabbit hole.

How to use this site

Accounting

You may already have noticed the Sign up and Log in buttons in the top right corner.

You can create a new account with the former, or log in with the latter by providing your user name or email address.

Once you are logged in, your user name will appear in the top right corner. Click it to quickly access your most recent programs, view your profile, or log out.

With an account, you can save and share your programs, and access them from multiple devices. That said, an account is not a requirement for creating programs on this site. They will however get lost forever as soon as you close the browser tab.

Creating and saving programs

The New button in the top left corner will open a blank program in the Paperbots IDE.

Inside the IDE, you will get an additional button called Save that lets you save your program (provided you are logged in). The Save is also available when viewing other people's projects. Clicking it will save a copy of that program into your account.

A quick IDE tour

Toolbar

At the top of the IDE, you find the buttons to create a new program, or save the current one. Next to these buttons, you can see the program's title, which you can edit. If the program belongs to another user, the toolbar will state the user name.

When you click "Save", the program's title, code, and the micro world setup will be saved, overwritting the last saved version of the program.

Code editor

The source code of your program is entered in the code editor (the panel labeled with PROGRAM). It will display errors detected by the compiler at the bottom of the code editor, and highlight the errornous part of the program.

The eye icon in the top right lets you enable and disable embedding of image and YouTube URLs inside the code editor.

You can click to the left of a line number to set a breakpoint for that line.

Help

The HELP panel shows all available functions and language syntax. You can collapse the panel by clicking on its header (the gray part saying "HELP"), to make more room for the code editor.

The micro world may provide additional functions on top of the built-in functions of the programming language. These will be shown at the top of the help panel.

Debugger

To the left of the code editor and help panel, you can see the collapsed debugger panel. Clicking on the green play button will run the program. Clicking the red bug button will start the program and pause it at the first statement that is going to be executed.

You can stop the program at any time by clicking the stop button.

You can pause the program at any time by clicking the pause button. The program will halt on the line that it will execute next.

When a program is paused, you can resume it by clicking the resume button.

When a program is paused, you can use the step over and step in buttons to step over the next statement, or, if the next statement calls a function, step into that function.

Finally, the debugger shows you all variables currently in scope, as well as the callstack. Clicking on a variable will highlight it in the code editor. Clicking on a frame in the callstack will highlight the statement the frame will execute next, and show the variables in scope.

You can not edit or save your program while it is running or paused. Only a stopped program can be saved.

Micro world - Robot land

To the right of the code editor, you find the micro world for which we write the program. Currently, only one micro world is supported: robot land. The program you write is actually a program for the robot that lives in this micro world. The robot provides functions to our program that we can use to control the robot. The functions are listed and documented in the help panel.

The toolbar at the top of the micro world lets you select a tool to modify the world. To use a tool, click it, then "draw" on the grid.

  • Robot: positions the robot on the grid. Clicking the robot itself will turn it.
  • Floor: deletes any walls, numbers or letters.
  • Wall: places walls.
  • Number: places a number between 0-99 on the grid.
  • Letter: places a letter on the grid.

You can use these tools to setup the world for your robot to execute its program in. While your program runs, it may modify the world. However, once the program has stopped, the world gets reset to the state it was in before the program was run.
